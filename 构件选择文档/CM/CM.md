#CM构件选择文档


在各组的组件中，我们主要看重的是否提供了和我们原有CM组件提供了**尽可能相似的接口**从而降低更改成本，在此基础上看文档在**使用方法上是否明确**，因此，综合比较后，我们选择了**第五组**的`CM`组件，

#### T1

+ 如果key已存在，setProperty(key)结果不明确
+ 没有带有地址参数的创建函数
+ 没有读取所有配置的接口
+ 没有更新版本的介绍

#### T3

+ 功能齐全
+ 有更新版本的文档
+ 没有带有地址参数的创建函数
+ 没有读取所有配置的接口

#### T5

+ 有示例代码
+ 有带有地址参数的创建函数
+ 有读取所有配置的接口
+ 有输入说明
+ **有和我们组类似的接口**
+ 有版本更新文档

#### T7

+ 版本更新频繁
+ 功能接口很多，但和我们的需求并不相同
+ 输入输出格式及注意事项非常明确
+ 使用`*.ini`文件作为配置文件，但是其他组多采用`*.properties`文件，本组也如此
